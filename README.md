# Truong Huy Research - Market Views

Astro portfolio site for valuation work, weekly market views, and monthly market views.
Market-view content is generated by Python scripts, scheduled by GitHub Actions, and
published through the normal site deployment flow.

Live: **https://truonghuyresearch.xyz**

---

## Project Structure

```text
/
+-- .github/workflows/
|   +-- daily_foreign_flow.yml      # 17:00 ICT daily data collection
|   +-- weekly_market_view.yml      # 17:00 ICT Friday weekly generation
|   +-- monthly_market_view.yml     # 17:00 ICT calendar month-end generation
+-- src/content/
|   +-- market-views/               # Weekly posts + rolling quarterly/memory JSON
|   +-- monthly-views/              # Monthly posts + monthly summary JSON
+-- weekly_bot.py                   # Weekly Market View generator
+-- monthly_bot.py                  # Monthly Market View generator
+-- market_memory.py                # Shared weekly/monthly/quarterly narrative state
+-- requirements.txt
+-- package.json
```

## Commands

| Command | Action |
| :-- | :-- |
| `npm install` | Install Astro + Tailwind dependencies |
| `npm run dev` | Local dev server at `http://localhost:4321` |
| `npm run build` | Build production site to `./dist/` |
| `npm run preview` | Preview the production build |
| `python weekly_bot.py` | Generate this week's Weekly Market View |
| `python weekly_bot.py --week 2026-06-05` | Generate for a specific Friday |
| `python weekly_bot.py --collect-foreign-flow` | Collect daily foreign-flow cache without LLM |
| `python monthly_bot.py` | Generate the previous Monthly Market View |
| `python monthly_bot.py --month 2026-05` | Generate for a specific month |

## Automation Schedule

All times below are configured as UTC cron equivalents for Asia/Saigon.

| Workflow | Local time | UTC cron | Purpose |
| :-- | :-- | :-- | :-- |
| `daily_foreign_flow.yml` | 17:00 ICT every day | `0 10 * * *` | Collect foreign-flow cache; no LLM |
| `weekly_market_view.yml` | 17:00 ICT every Friday | `0 10 * * 5` | Collect Friday data, generate weekly view, update quarterly + shared memory |
| `monthly_market_view.yml` | 17:00 ICT calendar month-end | `0 10 28-31 * *` plus month-end guard | Collect month-end data, generate monthly view, update monthly + shared memory |

`market_memory.py` links weekly, monthly, and quarterly narratives through
`src/content/market-views/_market_memory.json`. Weekly and monthly prompts both read
this shared state so new notes must reconcile continuation themes, thesis breaks,
technical levels, and forward risks.

## LLM Configuration

The bots are provider-neutral and use an OpenAI-compatible endpoint by default.

| Variable | Default | Notes |
| :-- | :-- | :-- |
| `LLM_API_KEY` | required | Primary hosted LLM API key |
| `OPENAI_API_KEY` | fallback | Accepted for local OpenAI-direct runs |
| `LLM_BASE_URL` | `https://api.openai.com/v1` | Optional OpenAI-compatible endpoint |
| `LLM_MODEL` | `gpt-5.2` | Optional model override |

GitHub Actions should store the key in `secrets.LLM_API_KEY`. Optional provider/model
overrides can be stored as repository variables `LLM_BASE_URL` and `LLM_MODEL`.
